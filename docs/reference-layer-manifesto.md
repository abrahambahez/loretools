# The Reference Layer. A Manifesto
created: 2026-04-17

The ideas we start from:

1. References are epistemic objects; that is, they build scalable networks of knowledge by iteratively documenting ideas, things, or events.
2. But to be such, they must be embedded in knowledge infrastructures that make them available to an epistemic public (communities, people, agents who think with and through them).
    - This implies that anything that cannot circulate through these networks cannot create knowledge from it.
3. There are as many knowledge frameworks as there are infrastructures that support publics and communities.
4. References are signs, and as such, they have potentially infinite interpretations that cause them to change over time.
5. While positive reference systems limit themselves only to evaluating the logic, "truth," and "objectivity" of references, expanded systems must include "honesty" (lying is not the same as being wrong), "intentionality" (being motivated by affects, passions, values, projects), and the "knowledge ecologies" in which they participate.

The promise of an expanded system:

1. Do not exclude references by default. Making available objects that are generally excluded by the strongest knowledge infrastructures allows for expansion.
2. Create referencing/documentation frameworks to include the greatest possible number of objects (gossip, ephemeral acts, bodily gestures, constructive practices passed from generation to generation), their retrieval media and, most critically, the knowledge frameworks that give them meaning.
3. Allow all references in a collection to participate equally in discussions relevant to their epistemic publics, assuming that contradiction is a necessary condition of the heterogeneity of the knowledge frameworks generated thanks to infrastructures.
